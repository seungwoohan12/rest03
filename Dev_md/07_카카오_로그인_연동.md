## 1. Supabase 카카오 OAuth 설정

| 항목 | 값 |
|------|----|
| Provider | Kakao |
| Client ID | 카카오 REST API 키 |
| Client Secret | REST API 키와 동일 값 입력 (보안 탭 미제공 앱에 대한 우회) |
| Redirect URI (카카오 앱 등록) | `https://bzdrzewkqxgogdebydld.supabase.co/auth/v1/callback` |

### Client Secret 관련 이슈

카카오 개발자 콘솔의 앱 유형에 따라 **보안 탭이 없어** Client Secret 코드가 생성되지 않는 경우 있음.

| 케이스 | 해결책 |
|--------|--------|
| 보안 탭 있음 | 코드 생성 후 Supabase에 입력 |
| 보안 탭 없음 (현재) | REST API 키를 Client Secret 칸에도 동일하게 입력 |

Supabase는 PKCE(Proof Key for Code Exchange) 방식을 기본 사용하므로 Client Secret이 없어도 동작 가능.

## 2. 코드 변경 — Login.jsx

### 카카오 로그인 버튼 추가

```jsx
async function signInWithKakao() {
  await supabase.auth.signInWithOAuth({
    provider: 'kakao',
    options: {
      redirectTo: 'https://seungwoohan12.github.io/rest03/',
    },
  })
}
```

- `redirectTo`: 카카오 인증 완료 후 돌아올 GitHub Pages URL
- 카카오 공식 브랜드 컬러 `#FEE500` 적용, 텍스트 `#3C1E1E`

### 로그인 페이지 구조 (수정 후)

```
[카카오로 로그인]  ← 카카오 황금색 버튼 (상단 강조)
─── 또는 이메일로 ───
[로그인 | 회원가입] 탭
  이메일 / 닉네임 / 비밀번호 입력
[로그인 or 회원가입]
```

## 4. 배포 방식 변경

GitHub Actions(`peaceiris/actions-gh-pages`) 배포가 간헐적으로 반영 안 되는 문제 발생.
→ **dist 폴더를 직접 gh-pages 브랜치에 force push** 하는 수동 배포 방식으로 전환.

```powershell
# 수동 배포 명령 (PowerShell)
$deployDir = "C:\Users\swhan\AppData\Local\Temp\hanu-deploy"
Remove-Item -Recurse -Force $deployDir
Copy-Item -Recurse "F:\rest03\dist" $deployDir
Set-Location $deployDir
git init && git checkout -b gh-pages
git add -A && git commit -m "Deploy"
git remote add origin "https://github.com/seungwoohan12/rest03"
git push --force origin gh-pages
```
