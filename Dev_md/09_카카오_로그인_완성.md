# 09. 카카오 로그인 트러블슈팅 및 완성

작업일: 2026-06-10

## 최종 동작 방식

`supabase.auth.signInWithOAuth` (리다이렉트) + `scope: 'profile_nickname'` 제한

---

## 트러블슈팅 전체 흐름

### 시도 1 — Kakao JS SDK v2 `Auth.login` 팝업 (#08)

```js
window.Kakao.Auth.login({ scope: 'openid,profile_nickname', success, fail })
```

**에러**: `window.Kakao.Auth.login is not a function`

**원인**: Kakao JS SDK v2에서 `Auth.login` 팝업 방식이 제거됨.
`t1.kakaocdn.net/kakao_js_sdk/2.7.2/kakao.min.js` (v2.7.2) 뿐 아니라
`developers.kakao.com/sdk/js/kakao.min.js` (unversioned) 도 현재 v2를 서빙.
v2의 `Kakao.Auth`에는 `authorize` (리다이렉트) 만 남아있고 `login` (팝업)은 없음.

---

### 시도 2 — `signInWithOAuth` 복귀 + scope 제한

```js
supabase.auth.signInWithOAuth({
  provider: 'kakao',
  options: {
    redirectTo: 'https://seungwoohan12.github.io/rest03/',
    scopes: 'profile_nickname',
  },
})
```

**에러**: `설정하지 않은 카카오 로그인 동의 항목을 포함해 인가 코드를 요청했습니다. 설정하지 않은 동의 항목: account_email, profile_image`

**원인**: Supabase Kakao 프로바이더가 기본적으로 `profile_nickname profile_image account_email` 세 가지 scope을 요청함.
개인 앱은 `account_email` / `profile_image` 동의항목에 권한이 없음 (비즈니스 앱 심사 필요).

**조치**: 카카오 개발자 콘솔 → 카카오 로그인 → 동의항목에서
`프로필 사진(profile_image)` → **선택 동의** 활성화

---

### 시도 3 — `account_email` 동의항목 설정 시도

카카오 개발자 콘솔에서 `카카오계정(이메일)` 동의항목을 선택 동의로 변경 시도.

**에러**: 권한 없음 (개인 앱은 email scope 비활성 — 비즈니스 앱 심사 필요)

**에러**: `설정하지 않은 동의 항목: account_email`

**원인**: `scopes: 'profile_nickname'` 을 코드에 추가했지만 브라우저가 이전 JS 번들을 캐시해서 새 코드가 로드되지 않음.

**조치**: 시크릿 창(Ctrl+Shift+N)에서 테스트 → 캐시 없이 새 코드 로드 → **정상 동작 확인**

## 최종 설정 요약

### 카카오 개발자 콘솔

| 항목 | 설정 |
|------|------|
| 카카오 로그인 활성화 | ON |
| Redirect URI | `https://[supabase-project-id].supabase.co/auth/v1/callback` |
| 동의항목 — 닉네임 | 필수 동의 |
| 동의항목 — 프로필 사진 | 선택 동의 |
| 동의항목 — 이메일 | 권한 없음 (개인 앱 제한) → 요청 안 함으로 처리 |
| 보안 탭 — Client Secret | 생성 후 Supabase에 등록 |

### Supabase Dashboard

| 항목 | 값 |
|------|-----|
| Provider | Kakao 활성화 |
| Client ID | 카카오 REST API 키 |
| Client Secret | 카카오 보안 탭에서 생성한 코드 |

### 코드 (Login.jsx)

```js
async function handleKakaoLogin() {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'kakao',
    options: {
      redirectTo: 'https://seungwoohan12.github.io/rest03/',
      scopes: 'profile_nickname',
    },
  })
  if (error) throw error
}
```

- `scopes: 'profile_nickname'` — 권한 없는 `account_email` 요청 차단
- 카카오 유저는 이메일 없이 카카오 고유 ID로 Supabase 인증
- 리다이렉트 완료 후 AuthContext가 세션 감지 → 자동 로그인 처리
